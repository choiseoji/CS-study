🔎 현재 User 테이블에 이름과 createdAt이 있습니다.

우리는 `19년 3월 01일` ~ `26년 03월0 1일` 사이의 `홍길동` 이름을 가진 유저를 찾고싶은데 인덱스를 어떻게 걸어야 할까?

## 1. 이름에만 인덱스를 넣는다면?

1. 인덱스에서 `홍길동`을 모두 찾습니다.

2. 찾은 `홍길동`에서 테이블로 일일이 찾아가 실제 `createdAt`이 조건에 맞는지 확인합니다.

3. 만약 `홍길동`이 1만 명이라면 (그럴일이 있나..?) 날짜 확인을 위해 테이블을 1만 번 확인해야 합니다.

## 2. 이름과 날짜가 같이 있다면?

1. 인덱스에서 `홍길동`을 모두 찾습니다.

2. 그 안에서 이미 정렬되어 있는 `createdAt`을 보고 조건에 맞는 범위만 고릅니다.

3. 조건에 맞는 결과가 10명이라면 테이블은 딱 10번만 확인하게 됩니다. (또는 SELECT 절에 다른 컬럼이 없다면 테이블을 안 확인해도 됨)

## 3. 만약 날짜 - 이름 순으로 복합 인덱스를 건다면?

이는 우리가 원하는 방향으로 흘러가지 않습니다.

먼저 `createdAt`부터 정렬되기 때문에 해당 날짜의 데이터를 찾는 것보다 2번째 방법으로 데이터를 골라내는 것이 효율적이기 때문입니다.
