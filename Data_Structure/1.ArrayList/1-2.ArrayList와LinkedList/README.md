## LinkedList 와 ArrayList 의 차이는 무엇인가요? 데이터 삽입, 삭제가 빈번하다면 무엇을 택해야할지 근거와 함께 설명해주세요.

ArrayList는 메모리에 연속적으로 데이터를 저장하는 동적 배열이며, LinkedList는 각 노드가 데이터와 이전/다음 포인터를 가지는 연결 리스트 구조입니다.

ArrayList는 인덱스를 통한 접근이 O(1)로 빠르지만, 중간 삽입/삭제 시 뒤의 요소들을 이동시켜야 하므로 O(n)의 시간이 발생합니다. 반면, LinkedList는 삽입/삭제 자체는 포인터 변경으로 O(1)이지만, 해당 위치를 찾는 과정이 필요하면 O(n)이 발생합니다.

따라서 삽입, 삭제의 위치를 이미 알고 있다고 가정하면 LinkedList가 더 적합할 것 같습니다.

</br>
</br>

**ArrayList**

- 동적 배열
- 데이터가 메모리에 연속적으로 저장됨
- 인덱스로 직접 접근 가능
- 데이터 삽입, 삭제 시에 뒤 요소들을 전부 이동시켜야 하기 때문에 O(n)이라는 시간 복잡도를 갖는다.

</br>

**LinkedList**

- 이중 연결 리스트
- 각 노드가 (데이터 + 이전/다음 포인터)를 가짐
- 메모리 상에서 흩어져 있음
- 삽입, 삭제 자체는 포인터 변경으로 처리되어 O(1)
- 하지만 해당 위치를 찾는 과정까지 포함된다면 O(n)

</br>

**데이터 삽입, 삭제가 빈번하다면??**

삽입, 삭제의 위치를 이미 알고 있다고 가정한다면 LinkedList가 더 적합하다.

왜냐하면 ArrayList는 중간 삽입, 삭제 시 뒤의 요소들의 위치를 전부 이동시켜야 해서 O(n)이라는 시간이 더 걸리지만, LinkedList는 해당 노드의 포인터만 변경하면 되기 때문이다.

</br>

**+) 하지만 ArrayList는 캐시 효율이 좋다**

ArrayList는 메모리가 연속적으로 배치되어 있기 때문에 CPU가 데이터를 읽을 때 공간 지역성이 좋아 캐시 적중률이 높다.

반면 LinkedList는 각 노드가 메모리 상에 흩어져 있어 포인터를 따라가며 접근하기 때문에 캐시 미스가 자주 발생하여 실제 성능에서는 ArrayList가 더 유리한 경우가 많다고 한다!
