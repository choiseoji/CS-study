# 어떠한 경우가 RESTful 하지 못한 것인지 설명해주세요. (꼬리질문)

***

#### REST API 설계 규칙

1. URI 는 명사를 사용해야 한다.

- `getAllUsers` 는 동사로 되어있다. ❎

- URI는 자원을 명시해야 하고 행위는 URI 가 아닌 HTTP 메서드로 정의하는 것이다.

위를 `GET /users` 로 정의하는 것이 올바르다. ✅

2. 슬래시(/)로 계층 관계를 표현하고

- `GET /user123Orders` ❎

-> `GET /users/123/orders` ✅

3. URL 마지막은 슬래시를 포함하지 않는다.

- `GET /users/` ✅

-> `GET /users` ❎

4. 밑줄(_)은 불가 하이픈(-) 가능

- `GET /best_selling-_tems` ❎

-> `GET /best-selling-items` ✅

5. URI 는 소문자만

- `GET /Users/UserProfile` ❎

-> `GET /users/user-profile` ✅

6. HTTP 응답 상태 코드 사용하고

- `200 Ok` 반환 
- 응답 본문 ❎

```
{
    "status": "error",
    "code": 404,
    "message": "User not found"
}
```

-> `404 Not Found` ✅

7. 파일 확장자는 URI 에 포함하지 않는다.

- `GET /users/123.json` ✅

-> 헤더에 application/json 을 포함해야 한다. ✅
