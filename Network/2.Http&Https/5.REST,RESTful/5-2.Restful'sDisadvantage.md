# RESTful 웹 서비스의 단점이 무엇인가요? (꼬리질문)

***

### 장점

1. HTTP 메서드와 URI 설계 규칙이 명확해 **표준화된 구조**

2. HTTP 기반이기에 **브라우저/서버 호환성**

3. GET 은 브라우저/프록시 서버에 **캐싱**이 가능

4. **Stateless** 구조이기에 확장성과 안정성


### 단점

1. 필요보다 더 많은 데이터를 받아야 하는 경우가 생긴다.

- 프로필 사진만 필요한데, 이름이나 나이 등등을 제공함

2. 한 화면을 구성하는 데 여러 API를 호출해야 하는 경우가 있다.

    BFF 를 찾아보면 좋음

3. 응답 구조가 고정되어 있어 유연하게 조절하기 어렵다.

- web 이나 app 에 따라 해상도 크기 차이로 app 은 더 적은 데이터만 필요할 수도 있다.

4. URI에 버전을 명시해야 하며, 관리가 번거로워진다.

### GraphQL

페이스북에서 개발한 API 쿼리 언어로 클라이언트가 필요한 데이터만 정확하게 요청할 수 있도록 설계된 API 방식이다.
