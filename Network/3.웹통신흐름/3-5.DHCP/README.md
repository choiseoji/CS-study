## DHCP란 무엇인가요?

DHCP는 네트워크에 연결된 장치에게 IP 주소와 네트워크 설정 정보를 자동으로 할당해주는 프로토콜입니다.

같은 네트워크에 연결된 장치들은 고유한 IP 주소가 필요한데, DHCP는 이를 자동으로 관리하여 IP 충돌을 방지하고 네트워크 관리 편의성을 높여줍니다.

일반적으로 가정에서는 공유기가 DHCP 서버 역할을 수행하며, IP 주소뿐만 아니라 서브넷 마스크, 기본 게이트웨이, DNS 서버 주소 등의 정보도 함께 제공합니다.

</br>
</br>

### DHCP란??

- Dynamic Host Configuration Protocol
- 네트워크 장치에 IP 주소와 네트워크 설정 정보를 자동으로 할당해주는 프로토콜이다. (잠깐 빌려주는 방식)
- 사용자가 직접 IP 주소를 설정하지 않아도 네트워크에 연결될 수 있도록 도와준다.
- 일반적인 가정 환경에서는 공유기가 DHCP 서버 역할을 수행한다.

</br>

**필요한 이유**

- 같은 네트워크에 연결된 장치들은 서로를 구분하기 위한 고유 IP 주소가 필요하다.
- 수동으로 IP를 설정할 경우 관리가 어렵고, 충돌이 발생할 수 있다.
- DHCP는 사용 가능한 IP주소를 자동으로 할당하여 이러한 문제를 해결한다.

</br>

### 공유기와 DHCP

- 공유기는 외부 네트워크과 내부 네트워크를 연결하는 장치이다. (인터넷 ↔ 공유기 ↔ 노트북, 휴대폰)
- 그래서 공유기는 2개의 IP주소를 가진다 ⇒ Public IP와 Private IP

</br>

**Public IP (공인 IP)**

- 인터넷에서 사용하는 주소
- 통신사(KT, SKT, LGU+ 등)가 할당
- 인터넷 전체에서 유일한 주소

</br>

**Private IP (사설 IP)**

- 내부 네트워크에서 사용하는 주소
- 공유기가 DHCP를 통해 각 장치에 할당

</br>

**DHCP가 하는 일**

- 같은 네트워크에 연결된 클라이언트에게 사용 가능한 사설 IP 주소를 할당한다.
    - 같은 네트워크 안에서 호스트 주소를 나누어서 할당
    
    ```sql
    노트북  → 192.168.0.100
    휴대폰  → 192.168.0.101
    태블릿  → 192.168.0.102
    ```
    
- 단순히 IP 주소만 할당하는 것이 아니라 아래와 같은 통신에 필요한 네트워크 설정 정보도 함께 전달한다.
    - IP 주소 : 기기가 사용할 IP 주소
    - 서브넷 마스크 : 같은 네트워크 범위를 식별하기 위한 정보
    - 기본 게이트웨이 : 외부 인터넷으로 나갈 때 사용할 장치 주소 (일반적으로 공유기 주소)
    - DNS 서버 주소 : 도메인을 IP 주소로 변환할 DNS 서버 주소
    - Lease Time : 현재 IP 주소를 사용할 수 있는 기간

공유기가 나와서 추가적인 개념

</br>

### NAT 란??

- Network Address Translation
- 사설 IP 주소를 공인 IP 주소로 변환하는 기술이다.
- 일반적으로 공유기에서 수행한다.
- 내부 네트워크의 장치들이 하나의 공인 IP를 통해 외부 인터넷과 통신할 수 있도록 한다.

</br>

**필요한 이유**

내부 네트워크에서는 사설 IP 주소를 사용하는데, 사설 IP 주소는 인터넷을 사용할 수 없다. 따라서 외부와 통신하기 위해서는 공인 IP로 변환이 필요하고 이를 NAT가 해준다.

**장점**

- IPv4 주소 절약
    - 여러 장치가 하나의 공인 IP를 공유할 수 있다.
- 보안성 향상
    - 외부에서는 내부 네트워크 구조를 직접 확인할 수 없다.
    - 사설 IP가 외부에 노출되지 않는다.

</br>

### DHCP와 NAT 차이

| DHCP | NAT |
| --- | --- |
| IP 주소 할당 | IP 주소 변환 |
| 내부 네트워크 관리 | 외부 인터넷 통신 지원 |
| "너는 이 IP 써" | "이 IP를 공인 IP로 바꿔줄게" |
