# 프로세스와 스레드의 차이 - 자원 격리와 공유

> - 결정적 차이는 주소 공간을 격리하느냐 공유하느냐의 차이
> - 프로세스는 독립된 주소 공간을 가져 서로 격리되고, 스레드는 프로세스의 주소 공간을 공유

## 핵심 비교

|       구분       |           프로세스           |            스레드            |
|:--------------:|:------------------------:|:-------------------------:|
|     주소 공간      |         독립 (격리)          |        프로세스 내에서 공유        |
|       자원       |   자기만의 메모리·파일 디스크립터 소유   |   프로세스 자원 공유, Stack만 고유   |
|     통신 방식      |  IPC (파이프, 소켓, 공유 메모리)   |    공유 메모리(Heap) 직접 접근     |
|     생성 비용      |    큼 (주소 공간·자원 복제 필요)    |   작음 (자원 공유, Stack만 할당)   |
| Context Switch | 큼 (페이지 테이블 교체 + TLB 플러시) |       작음 (주소 공간 유지)       |
|     장애 격리      | 한 프로세스 다운이 타 프로세스에 영향 없음 | 한 스레드 비정상 종료가 프로세스 전체에 영향 |

## 자원 격리 - 프로세스의 강점

프로세스는 독립된 주소 공간을 갖기에, 한 프로세스의 오류가 다른 프로세스의 메모리를 침범할 수 없다.

- 각 프로세스는 자기만의 페이지 테이블을 통해 별도의 물리 메모리에 매핑됨
- 한 프로세스가 잘못된 포인터로 메모리를 망가뜨려도 그 피해는 자기 주소 공간 안에 갇힘
- 한 프로세스가 크래시(예: Segmentation Fault)로 죽어도 다른 프로세스는 정상 동작

대신 통신 비용이 많이 발생하게 되는데, 서로 다른 주소 공간이라 데이터를 직접 주고받을 수 없어 커널을 경유하는 IPC가 필요하다.


## 자원 공유 - 스레드의 강점

스레드는 프로세스의 주소 공간을 공유하기에, 별도 장치 없이 같은 메모리를 통해 직접 데이터를 주고받는다.

- 같은 Heap을 공유하므로 한 스레드가 만든 객체를 다른 스레드가 즉시 참조 가능
- 통신을 위해 커널을 경유할 필요가 없어 빠름
- 생성·전환 시 복제·교체할 자원이 적어 비용이 작음

같은 메모리를 함께 쓰므로 한 스레드의 오류가 프로세스 전체로 번지고 동시 접근에 따른 동기화 문제가 생긴다.

## 장애 전파 범위의 차이

격리와 공유의 차이는 장애 전파 범위에 결정적 영향을 미친다.

|     상황     |        프로세스         |           스레드           |
|:----------:|:-------------------:|:-----------------------:|
| 하나가 비정상 종료 | 해당 프로세스만 종료, 나머지 정상 | 프로세스 전체가 종료 → 모든 스레드 소멸 |
| 메모리 오염 발생  |   자기 주소 공간 안에 격리    |  공유 메모리를 통해 다른 스레드에 전파  |

스레드는 주소 공간을 공유하므로 한 스레드에서 발생한 치명적 오류가 프로세스 전체에 퍼지지만, 프로세스는 격리된 주소 공간 덕분에 한 프로세스의 장애가 다른 프로세스에 영향을 주지 않는다.
