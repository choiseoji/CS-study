## 페이징(Paging) 기법의 개념과 이를 통해 외부 단편화를 해결하는 장점 및 내부 단편화가 생기는 단점을 설명해 주세요.

페이징(Paging)은 가상 메모리를 일정한 크기의 페이지(Page) 단위로 나누고, 물리 메모리(RAM) 역시 동일한 크기의 프레임(Frame) 단위로 나누어 관리하는 메모리 관리 기법입니다.

프로세스의 페이지들을 여러 프레임에 분산 저장할 수 있기 때문에, 연속된 물리 메모리 공간이 필요하지 않아 외부 단편화 문제를 해결할 수 있습니다.

다만 페이지 크기가 고정되어 있기 때문에 마지막 페이지에서 일부 공간이 남아 내부 단편화가 발생할 수 있다는 단점이 있습니다.

</br>
</br>

### 페이징

운영체제가 가상 메모리를 관리하기 위해 사용하는 메모리 관리 기법이다.

메모리를 일정한 크기의 페이지(Page) 단위로 나누어 관리하며, 실제 물리 메모리 역시 동일한 크기의 프레임(Frame)으로 나누어 사용한다.

프로세스의 페이지들은 연속된 물리 메모리에 저장될 필요 없이, 비어있는 프레임에 분산되어 적재될 수 있다.

</br>

### 외부 단편화 해결

- 메모리 공간은 충분하지만, 작은 빈 공간이 여기저기 흩어져 있어 연속된 큰 메모리를 할당하지 못하는 문제
- 페이징은 메모리를 고정 크기의 프레임 단위로 나누기 때문에 연속된 공간이 필요 없고, 빈 프레임만 있으면 어디든 할당 가능해서 외부 단편화가 발생하지 않음

</br>

### 내부 단편화 발생

- 할당된 블록 내부에서 공간이 남아 낭비되는 현상이다.
- 페이지 크기는 고정되어 있기 때문에 프로세스 크기가 정확히 페이지 크기의 배수로 나누어 떨어지지 않으면 마지막 페이지에 남는 공간 발생

</br>

### 그럼 OS는 왜 내부 단편화를 감수하고 페이징을 사용할까?

OS는 내부 단편화보다 외부 단편화 + 메모리 관리 복잡성 문제를 제거하는 것이 훨씬 중요하기 때문!!

외부 단편화가 더 치명적이다

- 총 빈 공간: 20MB
- 하지만 연속된 공간 없음 → 10MB 프로세스도 못 올림

메모리는 남아 있는데 못 쓰는 상태 → 시간이 지날수록 점점 상태 악화됨

그리고 연속 메모리 기반이면 OS가 할 일이 많아짐 → 다음 작업들을 수행해야 되는데 다 비용이 비쌈

- 어디에 넣을지 찾기 (first fit, best fit 등)
- 조각 관리
- 필요하면 압축(compaction)

그리고 페이지 크기가 고정이기 때문에 ‘내부 단편화’의 크기를 예측할 수 있음

(페이지 크기 4KB라면 낭비는 최대 4KB 미만)

또한, 프로세스 수십~수백 개이고 각 프로세스의 마지막 페이지 일부만 낭비하기 때문에 시스템 전체에서 보면 낭비가 작음
