## 만약 스택이 매우 커질 수 있으면 힙은 불필요할까요?

스택과 힙은 단순히 크기 차이가 아니라 메모리의 사용 목적 자체가 다르기 때문에 각각 필요하다고 생각합니다.

스택은 함수 호출 시 생성되는 지역 변수와 함수 실행 정보를 저장하며, 함수가 종료되면 자동으로 메모리가 해제됩니다. 따라서 수명이 함수 호출 범위를 벗어나는 데이터를 저장하기 어렵습니다.

반면 힙은 객체를 동적으로 생성하고, 여러 함수나 스레드가 공유할 수 있으며, 함수 종료와 무관하게 객체의 생명주기를 유지할 수 있습니다.

따라서 스택 크기가 매우 커지더라도, 장기간 유지되거나 공유되어야 하는 객체를 위해 힙 메모리는 여전히 필요합니다.

### Stack

- 함수의 지역 변수(자동 변수),  매개 변수, 반환 주소등이 사용하는 선형 메모리 공간
- LIFO 구조 → 중간 객체를 오래 유지하기 어려운 구조, 함수 범위를 벗어난 데이터 관리가 어려움
- Thread 별 독립적임 → 데이터를 공유하기 어려움
- 컴파일 타임에 소모될 스택 메모리의 크기를 알 수 있음 + 지역 변수의 수명을 추적할 수 있음
- 호출 스택이 무한정으로 늘어나는 것은 논리적으로 비효율적인 코드

</br>

### 스택과 멀티스레딩 이슈

- **스택에 특정 함수 범위를 넘어서도 존재해야 하는 변수(인스턴스)를 관리하기 어려움**
- 멀티스레딩 환경에서 **여러 스레드가 동시에 접근해야 할 필요성이 있는 대상 메모리가 필요한 경우** 구현이 어렵거나 구조적 불가능
