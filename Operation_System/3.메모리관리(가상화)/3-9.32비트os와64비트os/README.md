## 32bit os랑 64bit os랑 메모리 관점에서 어떤 차이가 있나요?

가장 큰 차이는 CPU가 표현할 수 있는 메모리 주소 공간 크기입니다.

32bit 환경은 약 4GB 정도의 주소 공간만 표현할 수 있지만, 64bit 환경은 훨씬 큰 주소 공간을 사용할 수 있어 대용량 메모리를 효율적으로 관리할 수 있습니다.

또한 주소 공간이 커지면서 관리해야 할 페이지 수도 증가하기 때문에, 64bit 환경에서는 더 많은 단계의 페이지 테이블 구조를 사용합니다

</br>
</br>

**메모리 주소 지정 가능 범위**

- 32 bit
    - 최대 메모리 주소 지정 : 2 ^ 32 = 4GB
    - 물리적으로 4GB 이상의 메모리를 인식하지 못 함
- 64 bit
    - 최대 메모리 주소 지정 : 2 ^ 64 = 16EB (엑시바이트)
    - 이론적으로는 매우 큰 메모리를 관리할 수 있지만, 실제로는 하드웨어나 os에 의해 제한됨

</br>

**페이지 테이블 관리**

가상 메모리에서는 논리 주소와 물리 주소를 연결하기 위해 페이지 테이블을 사용한다.

주소 공간이 커질수록 관리해야 할 페이지 수도 증가하므로, 64bit 환경에서는 페이지 테이블 크기도 훨씬 커진다.

32bit OS 에서 페이지 크기가 4KB 라고 했을 때

- 가상 주소 공간의 크기 : 2^32=4GB
- 페이지 하나 크기 : 2^12GB
- 가상 주소 공간 안의 페이지 수 : 2^20 = 약 100만개 페이지 = PTE도 약 100만개

그럼 64bit OS는?? 페이지 테이블 크기 엄청 커짐 → 페이지 테이블 자체가 메모리를 많이 차지하게 됨

그래서 **다단계 페이지 테이블 사용**

```
1단계 테이블
  ↓
2단계 테이블
  ↓
3단계 테이블
  ↓
실제 페이지
```

필요한 주소 영역에 대해서만 하위 페이지 테이블을 생성하여 메모리 사용량을 줄인다.

32bit OS도 다단계 페이지 테이블을 사용하지만, 64bit 환경에서는 주소 공간이 훨씬 크기 때문에 더 많은 단계의 페이지 테이블 구조를 사용한다.
