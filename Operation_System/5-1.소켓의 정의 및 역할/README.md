## 네트워크 통신의 종착점 역할을 하는 소켓(Socket)의 운영체제 레벨에서의 정의와 역할은 무엇인가요?
>소켓은 애플리케이션이 네트워크를 직접 다루지 않고 OS 커널의 TCP/IP 기능을 파일처럼 사용할 수 있게 해주는 추상화된 인터페이스입니다. 
> 
> 프로세스는 소켓을 파일 디스크립터로 참조하고, send나 recv 같은 시스템 콜을 통해 데이터를 보내거나 받습니다. 
> 
> 이때 실제 데이터는 커널의 송수신 버퍼와 TCP/IP 스택을 거쳐 처리됩니다. 
> 
> 따라서 소켓은 프로세스와 커널 사이의 네트워크 인터페이스이면서, 통신 상대를 식별하고 데이터를 주고받는 네트워크 통신의 종착점 역할을 합니다.

### 소켓
> 네트워크로 연결되어 있는 컴퓨터의 통신의 접정에 위치한 통신 객체

#### 그러나, OS 관점에서는...
소켓도 파일이다!

#### 파일
1. 일반 파일: SSD/HDD에 저장된 데이터 덩어리
2. 디렉터리 파일: 파일 이름과 위치 정보를 담는 파일
3. 장치 파일: 장치를 파일처럼 다룰 수 있게 해주는 특수 파일

소켓은 파일에 대한 접근을 위해 OS 커널에 구현되어 있는 프로토콜 요소를 추상화한 인터페이스

-> 내부적으로는 파일 디스크립터로 관리 / 파일처럼 시스템 콜로 다룰 수 있음

#### 즉, 프로세스가 커널의 TCP/IP 네트워크 기능을 사용하기 위해 제공받는 파일 디스크립터 기반의 통신 인터페이스

**[내부 동작 과정]**
![](https://velog.velcdn.com/images/leekhy02/post/1ec9644e-810c-4fea-b052-a9e4b3bbfda7/image.png)

Process: 우리의 애플리케이션 
```java
Socket socket = new Socket("example.com", 80);
```

이런 코드를 작성했으면

```text
프로세스 → OS 커널에게 socket 생성 요청
커널 → 소켓 커널 객체 생성
커널 → 이 소켓을 가리키는 fd를 프로세스에게 반환
```
그리고 프로세스가 OS에게 연결 요청

> "3번 fd가 가리키는 소켓을 example.com의 80번 포트에 TCP 연결해줘"

그럼 커널은 fd보고 연결

그리고 그 후
```java
socket.getOutputStream().write(data);
```

이렇게 썼다면 TCP/IP 계층으로 데이터 전달

_cf) TCP/IP, 드라이버 같은 핵심 네트워크 기능은 Kernel mode에 있기 때문에 프로세스가 네트워크 I/O를 하려면 시스템 콜을 통해 커널에 요청 필요 -> 그래서 소켓이 User mode와 Kernel mode 경계 근처에 위치_

### 소켓의 송수신
![](https://user-images.githubusercontent.com/37354145/124682309-6d820480-df05-11eb-92f4-f5937c710c53.png)


#### 클라이언트 흐름
1. 소켓 생성
2. 서버에게 TCP 연결 요청

#### 서버 흐름
1. 소켓 생성
2. IP 주소와 포트 번호 붙임
3. 연결 요청을 받을 수 있는 상태로 변화
4. 클라이언트 요청을 기다리다가 실제 연결 요청이 오면 연결 수락
   - server_fd: 연결 요청을 기다리는 리스닝 소켓
   - client_fd: 실제 클라이언트와 데이터를 주고받는 연결 소켓
   
    => accept(): server_fd로 통신하는게 아니라, 새로운 connected socket fd 반환하고 서버 프로세스가 이를 통해 송수신 진행

#### 송수신 과정
- 서버가 send() 호출
  
    서버 프로세스는 OS에게 "client_fd가 가르키는 TCP로 데이터 보내줘"라고 요청

    TCP 송신 버퍼에 데이터 저장 후 전송

    클라이언트 쪽에서는 socket의 수신 버퍼에 저장하고 클라이언트가 recv() 호출하면 그제서야 버퍼에서 데이터 복사 후 클라이언트 프로세스가 데이터 읽음

_cf) TCP는 양방향 통신이기 때문에 서버, 클라이언트 양방향으로 송수신 가능_

### 소켓 역할
1. 프로세스와 OS 커널 사이의 네트워크 인터페이스
2. 통신 상대를 식별하는 종착점 역할
3. 데이터 송수신을 위한 버퍼
