## **Q. Spring MVC 패턴에서 관심사의 분리가 무엇이고 어떻게 적용되는지?? Model과 View가 강결합되었을 때의 문제점**

관심사의 분리란 애플리케이션의 역할과 책임을 기능별로 분리하여 각 계층이 하나의 책임에만 집중하도록 설계하는 개념입니다.

Spring MVC에서는 이를 MVC 패턴으로 적용합니다.

- Controller는 요청 처리와 흐름 제어,
- Model은 데이터와 비즈니스 로직,
- View는 화면 렌더링을 담당합니다.

과거에는 하나의 Servlet에서 요청 처리, 비즈니스 로직, HTML 생성까지 모두 처리하는 경우가 많았는데, 이처럼 역할이 강하게 결합되면 화면 변경이 비즈니스 로직 수정으로 이어져 유지보수가 어려워집니다.

Spring MVC는 역할을 분리함으로써 코드 재사용성과 유지보수성을 높이고, 개발자가 각 계층의 책임에 집중할 수 있도록 합니다.

</br>
</br>

### 관심사의 분리(Separation of Concerns)란?

애플리케이션의 역할과 책임을 기능별로 분리하여 각 계층이 하나의 책임에만 집중하도록 설계하는 개념이다.

이를 통해

- 유지보수성 향상
- 코드 재사용성 증가
- 변경 영향 최소화

등의 장점을 얻을 수 있다.

</br>

### Spring MVC에서의 관심사 분리

Spring MVC에서는 MVC 패턴을 통해 관심사를 분리한다.

- **Controller**
    - 클라이언트 요청 처리
    - 요청 흐름 제어
- **Model**
    - 데이터와 비즈니스 로직 처리
- **View**
    - 사용자에게 보여지는 화면 렌더링

⇒ 즉 요청 처리, 비즈니스 로직, 화면 생성을 각각 분리하여 관리한다.

</br>

### Model과 View가 강결합되었을 때 문제점

과거 Servlet/JSP 기반 구조에서는 하나의 클래스에서

- DB 조회
- 비즈니스 로직
- HTML 생성

을 모두 처리하는 경우가 많았다.

이 경우

- 화면 변경이 비즈니스 로직 수정으로 이어질 수 있고
- 코드 재사용성이 낮아지며
- 유지보수가 어려워진다.

예를 들어 HTML 구조 변경만 필요해도 Servlet 코드를 수정해야 하는 문제가 발생한다.

MVC 패턴은 이러한 강결합 문제를 해결하기 위해 역할을 분리한 구조이다.
